apply: manually

1. 情報収集と分析
   あなたが以下の情報を自動的に収集・分析する。bashコマンドで各情報を取得する。bashコマンド以外は使わない。

    ```bash
    git branch --show-current
    git diff main
    ```

2. GitHub Issueの確認
   チケット番号が存在する場合、Issueを確認するため GitHub MCP Serverの `issue_read` コマンドを使用してissueの内容を取得すること
   チケット番号はブランチの名前の先頭にある数字列とする。

2. ブランチの変更内容の確認
   現在のブランチにコミットされている内容を確認し、あなたがレビューする。
   コミットされていないファイルに関しては無視する。

3. 結果報告
   レビュー内容をまとめて、下記のテンプレートでユーザーに報告する。
    ```
    ファイルのフルパス:行数
    内容
    ```
