# Track Specification: Issue #136 - 起動時の自動フォーカス設定

## 1. 概要 (Overview)
EmEditor プラグインが起動（表示）された際、ユーザーが即座にターミナル入力を開始できるよう、自動的にターミナルウィンドウ（ConPTY/GUI制御下）へフォーカスを遷移させる。
本仕様は、Issue #61（起動中の再実行クラッシュ）が先行して解消されていることを前提とする。

## 2. 機能要件 (Functional Requirements)
- **依存関係の成立**: Issue #61 の受け入れ条件（再実行時にクラッシュせず既存ターミナルへフォーカスできること）が満たされていること。
- **自動フォーカス設定**: `OnCommand` によるプラグインバー表示処理が成功した都度、`SetFocus` API を使用してターミナルペインへフォーカスを移す。
- **フォーカス抑止条件**: EmEditor 側でメニューまたはモーダルダイアログが表示中の場合は、フォーカス奪取を行わない。
- **Input モデル整合**: フォーカス遷移変更が `Input` ドメインモデルおよび既存キーボードフック挙動に影響を与えないことを確認する。

## 3. 非機能要件 (Non-Functional Requirements)
- **パフォーマンス**: フォーカス遷移に伴う目に見える遅延を最小限にする。
- **堅牢性**: 何らかの理由でフォーカス設定が失敗した場合でも、致命的なエラー（クラッシュやフリーズ）を引き起こさず、静かに失敗する（Silent Failure）。
- **回帰防止**: Issue #61 で解消した `OnCommand` 再実行時クラッシュを再発させない。

## 4. 受け入れ条件 (Acceptance Criteria)
- Issue #61 の受け入れ条件が満たされた環境で検証していること。
- プラグインバーを表示した直後に、マウスでクリックすることなく、キーボード入力がターミナルに反映されること。
- `OnCommand` 実行後の表示成功時に `SetFocus` が呼び出されていることをログまたは動作で確認できること。
- EmEditor のメニューまたはモーダルダイアログ表示中は、フォーカス奪取を行わないこと。

## 5. スコープ外 (Out of Scope)
- Issue #61 の根本修正（クラッシュ原因の特定および排他制御修正）そのもの。
- 他のプラグインバーやエディタ本体とのフォーカス切り替えに伴う詳細な挙動（Tab キー等）。
- コンテキストメニュー（右クリック）使用後のフォーカス復旧。
