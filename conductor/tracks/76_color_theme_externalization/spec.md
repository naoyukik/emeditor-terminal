# 仕様書: カラーテーマ設定の外部化とファイルからのロード機能 (Issue #76)

## 1. 概要 (Overview)
本タスクは、現在ハードコードされているターミナルのカラーテーマを外部設定（INI/レジストリ）からロードし、設定ダイアログのGUIからユーザーが切り替えられるようにする機能拡張である。
Epic #73 の一環として、ユーザーごとに最適な視認性（Dark/Light等のテーマ）を提供することを目的とする。

## 2. 機能要件 (Functional Requirements)
1. **設定値の永続化**:
   - `TerminalConfig` の `ThemeType` を、EmEditor プラグイン設定の `Terminal` セクションに保存・復元できるようにする。
   - `EmEditorConfigRepositoryImpl` を拡張し、テーマ設定の読み書きロジックを実装する。
2. **GUI設定ダイアログへの統合**:
   - プラグインの設定ダイアログに「カラーテーマ」を選択するUI（コンボボックス等を想定）を追加する。
   - 選択肢は以下の3つとする:
     - `System Default (Auto)` : EmEditor（またはOS）のテーマ設定に追従する（新規追加）
     - `One Half Dark` : 既存のダークテーマ
     - `One Half Light` : 既存のライトテーマ
3. **テーマの適用タイミング**:
   - 設定ダイアログでの変更は**再起動時（次回ターミナル起動時）**に反映される仕様とする（リアルタイム反映は本タスクのスコープ外）。
4. **`System Default` の動作定義**:
   - `System Default` が選択された場合、起動時にEmEditor自体のテーマ（またはOS設定）を判定し、DarkかLightかを自動で解決して描画に反映する。

## 3. 非機能要件 (Non-Functional Requirements)
- EmEditor SDKの `EE_REG_QUERY_VALUE` / `EE_REG_SET_VALUE` を用いて、既存のフォント設定等と同様の方式で永続化を行うこと。
- `ThemeType` の追加に伴い、下位互換性（既存設定ファイルにキーが存在しない場合のデフォルト値フォールバック）を担保すること。デフォルトは `System Default` を推奨。

## 4. 受け入れ基準 (Acceptance Criteria)
- 設定ダイアログからカラーテーマ（Auto, Dark, Light）を選択して「OK」を押せること。
- 選択した設定が再起動後に保持されていること（INI/レジストリに保存されていること）。
- ターミナルを再起動（またはプラグインをON/OFF）した際、選択したテーマの色でターミナルが描画されること。
- `System Default` 選択時、環境に応じた適切なテーマ（DarkまたはLight）が自動で選ばれること。

## 5. 対象外 (Out of Scope)
- テーマ変更を「設定ダイアログのOKボタン押下時」にリアルタイムで画面へ反映する処理。
- 新規のカスタムテーマ（ユーザー定義の色パレット等）の追加。本タスクは既存テーマの選択と永続化に留める。
