# Specification: Fix Issue 135 (Terminal text overflow at startup)

## Overview
ターミナル起動時（特に Gemini CLI 起動直後）に、物理的なウィンドウサイズと ConPTY に報告される論理サイズ（カラム数・行数）が一致せず、テキストが右端で折り返されずに突き抜ける、および画面下部にはみ出す不具合を修正する。このトラックは GitHub Issue 135 の元のタスクと、今回追加報告された下部のはみ出し症状の双方を解決する。

## Functional Requirements
- プラグイン起動直後から、ターミナルの論理サイズ（Cols x Rows）が物理的なクライアント領域と完全に一致していること。
- ウィンドウの右端でテキストが正しく折り返されること。
- ウィンドウの下部でテキストが見切れないこと（行数の計算が正しいこと）。
- 初期化シーケンスにおいて、ConPTY が描画を開始する前に正確な寸法が提供される、もしくは生成直後に即座に正しいサイズへリサイズされること。

## Proposed Approaches
1. **[推奨案] 起動時サイズの厳密な計算**:
   ウィンドウ作成時（`WM_CREATE` などの初期化時）に、フォントメトリクスとクライアント領域の正確なピクセルサイズを算出し、ConPTY の初期起動サイズとして渡す。
2. **[代替案] サイズ確定まで初期化を遅延**:
   初期起動時のサイズが不確実な場合、OS から確定した `WM_SIZE` メッセージを受信するまで ConPTY の起動とターミナルグリッドの初期化を待機させる。

## Environment Constraints
今回の不具合が顕著に発生している環境（テストケースとして使用）：
- FontFaceName: UDEV Gothic 35JPDOC
- FontSize: 14
- FontWeight: 400
- FontItalic: 0
- ColorTheme: 0

## Acceptance Criteria
- ターミナルを起動し Gemini CLI を実行した際、右端でテキストが正しく折り返されること。
- 起動直後、画面下部にテキストが隠れず、最後の行まで正常に表示されること。
- ウィンドウをリサイズした際にも、テキストが正常に追従・再レンダリングされること。
