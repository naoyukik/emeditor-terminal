# Specification: Issue 78 - 完全な自動保存の実装

## 1. Overview
EmEditor プラグインの設定（フォント、カラーテーマ、シェルパス）を、設定変更時およびプラグイン終了時に永続化領域（レジストリまたは INI ファイル）へ自動的に保存し、次回起動時に設定を復元できるようにする。

## 2. Functional Requirements
- **保存のトリガー**:
    - 設定ダイアログ等で設定が更新され、確定されたタイミング。
    - プラグインウィンドウが破棄されるタイミング（`WM_DESTROY` / `on_destroy`）。
- **保存対象データ**:
    - カラーテーマ (`ThemeType`)
    - フォント名 (`font_face`)
    - フォントサイズ (`font_size`)
    - フォントスタイル（太字・斜体） (`font_weight`, `font_italic`)
    - 使用するシェルのパス (`shell_path`)
- **保存先**:
    - EmEditor のプラグイン設定領域（`ConfigurationRepository` を介したレジストリまたは INI ファイル）。
- **エラーハンドリング**:
    - 書き込み権限の不足等で保存に失敗した場合、EmEditor のダイアログを表示してユーザーに通知する。

## 3. Non-Functional Requirements
- **パフォーマンス**:
    - 保存処理は軽量であるため、UI スレッドをブロックしても許容範囲内とする（EmEditor SDK の仕様に従う）。
- **堅牢性**:
    - HWND が無効な場合や終了処理の最中でも、クラッシュせずに安全に終了・保存を試みる。

## 4. Acceptance Criteria
- 設定ダイアログで設定を変更し、プラグインを再起動しても設定が維持されていること。
- プラグインウィンドウを閉じた際の設定が、次回の起動時に反映されていること。
- 保存失敗時に、ユーザーが認識できるダイアログが表示されること。

## 5. Out of Scope
- 本機能以外の設定（履歴等）の保存。
- クラッシュ時のリカバリ。
