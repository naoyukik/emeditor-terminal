# Specification - Issue #110: プラグイン設定ダイアログの実装

## 1. Overview
Issue #74 で実装された設定管理基盤 (`ConfigRepository`) を利用し、ユーザーが GUI 上で設定（主にフォント）を変更・保存できる「プラグイン設定ダイアログ」を実装する。EmEditor 本体の「プラグインの設定」画面からシームレスに呼び出し可能とする。

## 2. Functional Requirements
- **EmEditor 連携**:
    - `EP_QUERY_PROPERTIES` メッセージに対して `TRUE` を返し、設定ボタンを有効化する。
    - `EP_SET_PROPERTIES` メッセージ受信時に、モーダルな設定ダイアログを表示する。
- **設定項目 (Scope: Font Only)**:
    - **Font Name**: 現在使用中のフォント名を表示し、変更可能とする。
    - **Font Size**: フォントサイズを表示・変更可能とする。
    - **Font Style**: 太字、斜体等のスタイル情報を保持・変更可能とする。
- **フォント選択 UI**:
    - Windows 標準の `ChooseFontW` ダイアログを呼び出し、ユーザーが視覚的にフォントを選択できるようにする。
- **永続化**:
    - 「OK」ボタン押下時に、選択された内容を `ConfigRepository` を通じてレジストリ/INI に保存する。
- **反映タイミング**:
    - 本トラックでは「再起動後に反映」とする（実装の簡略化と安定性を優先）。変更を保存した際、ユーザーに再起動が必要な旨を通知するか、ダイアログに注釈を入れる。

## 3. Non-Functional Requirements
- **UI Language**: 英語 (English) のみで実装する。将来の多言語化に対応しやすいよう、リソース管理を行う。
- **Visual Style**: Windows 標準のコントロールを使用し、EmEditor 本体の UI と親和性の高い外観にする。
- **Error Handling**: フォント読み込み失敗時などにデフォルト設定へフォールバックする堅牢性を備える。

## 4. Acceptance Criteria
- [ ] EmEditor のプラグイン一覧で「ターミナル」の設定ボタンが有効になっている。
- [ ] 設定ボタンを押すと、フォント設定を含むダイアログが表示される。
- [ ] 「Change Font」ボタン等から標準フォント選択ダイアログが開き、フォントを変更できる。
- [ ] ダイアログで「OK」を押すと設定が保存され、再度ダイアログを開いたときに変更内容が保持されている。
- [ ] プラグイン（EmEditor）を再起動した際、新しい設定に基づいてターミナルが描画される。

## 5. Out of Scope
- シェルパスの設定、作業ディレクトリの設定（これらは将来のトラックで対応）。
- 設定のリアルタイム反映（動的なフォント変更）。
- リソーススクリプト以外の動的な UI 構築（基本は .rc を使用）。
