# 仕様書 (Specification): リサイズ・起動時のフォント設定リセット問題の修正

## 1. 概要 (Overview)
ターミナルプラグインの起動時やリサイズ時（`on_size`）において、ユーザーが定義したフォント設定（フォント名、サイズ、ウェイト）が正しく読み込まれない、あるいは維持されない問題が発生している。ログによれば、初回ロード直後に `font_size=0` および `weight=0` として設定が再ロードされており、ユーザーの設定値が無視されている。

## 2. 機能要件 (Functional Requirements)
- ターミナル起動時に、ユーザーのフォント設定（フォント名、サイズ、ウェイト）を正確に読み込むこと。
- ウィンドウのリサイズ（`on_size`）等の後続イベントにおいても、フォント設定が正しく維持されること。
- フォントサイズやウェイトが読み込めない（`0` に評価されるなど）場合、ターミナルの描画コンテキストに `0` を適用するのではなく、デフォルト値へのフォールバックやバリデーション処理を実装すること。

## 3. 非機能要件 (Non-Functional Requirements)
- **パフォーマンス (Performance)**: フォント設定の読み込みや適用によって、ターミナル起動時やリサイズ時に目に見える遅延が生じないこと。
- **安定性 (Stability)**: フォントサイズに依存するメトリクス計算が破損しないよう保証すること。

## 4. 受け入れ条件 (Acceptance Criteria)
- [ ] ターミナルプラグインの起動時に、ユーザーが設定したフォント名、サイズ、ウェイトが適用されること。
- [ ] ターミナルウィンドウをリサイズしても、フォントサイズやウェイトが `0` にリセットされないこと。
- [ ] ターミナルセッション中、有効なフォント設定（サイズ > 0, ウェイト > 0）がロードされ維持されていることがログから確認できること。
- [ ] フォントメトリクス計算の修正により、IMEの表示位置や一般的なテキスト描画にデグレ（退行バグ）が発生しないこと。

## 5. スコープ外 (Out of Scope)
- まったく新しいフォント描画システムの実装。
- 今回のフォントサイズ/ウェイト読み込み問題に直接関係しない、複雑な書記素クラスター（Grapheme Clusters）の対応。
