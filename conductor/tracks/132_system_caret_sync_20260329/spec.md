# Specification: システムキャレットの常時同期による IME 候補ウィンドウ表示位置の解決

## 1. 概要 (Overview)
Gemini CLI 等、複雑なカーソル移動を行う TUI アプリケーションにおいて、IME 候補ウィンドウが不適切な位置（画面外 0,0 等）に表示される問題を解決する。
現在の「IME 入力開始時のみ同期」という受動的なアプローチを、仮想カーソル移動に合わせた「能動的なシステムキャレット同期」へと刷新する。

## 2. 機能要件 (Functional Requirements)
- **能動的同期プロトコル**: 仮想カーソルの座標が変更されるすべてのタイミング（文字描画、CSI による移動、スクロール、Save/Restore）において、OS の `SetCaretPos` を呼び出す。
- **不可視キャレットの活用**: 自前描画カーソルとの描画干渉を防ぐため、`CreateCaret` にて幅0（または 1x1 の不可視設定）のキャレットを作成・同期する。
- **座標変換ロジックの統合**: `ConfigGuiDriver` や `TerminalGuiDriver` 等で個別に計算されていた IME 座標計算を、画面上の物理座標への一方向変換ロジックとして整理し、冗長な計算を排除する。
- **上位レイヤーへの影響最小化**: `TerminalWorkflow` (Domain) は「カーソルが移動した」という事実のみを管理し、実際の Win32 API 呼び出しは `GuiDriver` 層の隠蔽されたメソッドで行う。

## 3. 非機能要件 (Non-Functional Requirements)
- **パフォーマンス**: 高速なテキスト出力（`cat` 等）時でも、`SetCaretPos` の呼び出しが描画パフォーマンスを著しく低下させないこと。
- **堅牢性**: キャレットのリソース管理（`CreateCaret`, `DestroyCaret`）を RAII パターンに基づき安全に行う。

## 4. 完了条件 (Acceptance Criteria)
- [ ] Gemini CLI で日本語入力中、候補ウィンドウが常に仮想カーソルの直下に表示されること。
- [ ] Vim や PowerShell (PSReadLine) 等、他の TUI 環境での動作に退行（Regressions）がないこと。
- [ ] 自前描画のブロック型/バー型カーソルと、システムキャレットが二重に表示されたり、フリッカーが発生したりしないこと。

## 5. スコープ外 (Out of Scope)
- フォントごとの詳細な IME 候補ウィンドウのフォントスタイル同期（`ImmSetCompositionFont` の高度な設定）。
- マルチモニター環境における DPI スケーリングに起因する特殊な座標ずれ（既存の拡大率処理の範囲内で対応）。
