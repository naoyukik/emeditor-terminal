# Initial Concept
EmEditorにはターミナルを表示するプラグインや機能が存在していない。そのため、EmEditor内でターミナルを表示するプラグインを実装するプロジェクトです。

# Product Guide - emeditor-terminal

## Vision
EmEditor 内に統合されたターミナル機能を提供し、エディタを離れることなくコマンドライン操作を完結させる。特に AI エージェントを利用した開発において、コードの変更とその結果の確認をシームレスに行える環境を構築する。

## Target Users
- AI エージェントを CLI で稼働させ、その変更をリアルタイムで確認したいエンジニア
- EmEditor をメインの開発環境として利用しているプロフェッショナル

## Key Features
- **即時起動ターミナル**: プラグインボタンを押すだけで即座にデフォルトシェル（pwsh.exe）が起動し、自動的にターミナルへフォーカスが当たるため、即座に入力を開始できる。
- **統合 UI**: EmEditor のウィンドウ内にドッキングし、コードエディタとターミナルを同時に視認・操作可能。
- **AI 開発最適化**: AI によるコード修正を即座にターミナル上のコマンド（テスト実行など）で検証できるワークフロー。
- **テーマ対応**: モダンなカラーパレット（One Half Dark/Light）をサポートし、視認性を向上。自動コントラスト調整機能を備える。
- **カスタマイズ可能な UI**: フォント名やサイズなどの設定を GUI から変更可能。EmEditor 標準のプラグイン設定インターフェースに統合されている。
- **堅牢なウィンドウ制御**: 二重起動防止機能を備え、すでに開いている場合は既存のウィンドウにフォーカスを当てる。

## Goals
- EmEditor ユーザーに標準的なターミナル体験を提供する。
- AI エージェントとの協調開発における「コンテキストスイッチ」のコストを最小化する。

## Current Status (2026-04-18)
- **Advanced TUI Support**: Microsoft Edit 等のモダンな TUI アプリケーションの描画に対応。
  - **最新基盤への刷新**: `windows-rs` 0.62 (Release 73) へのアップデートを完遂。細分化された公式クレート群への移行により、将来的な保守性と型安全性を確保。
  - **アーキテクチャの完成 (Strict Rigid)**: プレゼンテーション層 (Resolver) から Win32 型を完全に抹消し、`WindowId` に統一。ドメインロジックの純粋化と物理的隔離を最高レベルで達成。
  - **スケーラブルなバッファ管理**: 肥大化したバッファ管理を `Grid` と `Scrollback` に分離し、コードの凝集度を向上。
  - **鉄壁の安全性ドキュメント**: すべての `unsafe` ブロックに対し、RAII やスレッド安全性に関する根拠を明文化。
  - **究極の表示整合性**: **Grapheme Clusters (書記素クラスター) 方式**への刷新により、結合文字や Emoji ZWJ シーケンス、国旗記号等が混在する環境下でも、物理カラムとの完全な同期を実現。
  - **インテリジェント・バッファリング**: 確定待ちバッファによる動的なクラスター判定により、複雑な Unicode シーケンスを正確に1つの論理文字として統合。
  - **物理グリッドの鉄壁保護**: 文字操作の前後でワイド文字の境界を自動修復し、長年の課題であった座標ズレを根絶。
  - **拡張色・スタイルサポート**: SGR シーケンスの完全サポートにより、256色、TrueColor (RGB)、太字、斜体、下線、取り消し線、反転の描画に対応。
  - **カラーテーマ**: One Half Dark/Light テーマをサポート。PowerShell等のディレクトリ表示におけるコントラスト問題を自動補正。
  - **高度な画面制御**: `DECSTBM` (スクロール領域制限), `DECOM` (原点モード), `RI/IND` (逆・順スクロール), `IL/DL` (行挿入・削除) 等の TUI 標準命令を網羅。
  - **BCE (Background Color Erase)**: スクロールやクリア操作時に現在の背景色を維持し、視覚的な一貫性を確保。
  - **Altキー操作**: `Alt` キーによるメニュー操作（Metaキー送信）をサポート。EmEditor 本体のメニュー起動を抑制し、TUI 内の操作を優先。
  - **描画安定性**: ダブルバッファリング（Memory DC）の導入により、高速更新時もフリッカー（ちらつき）のない滑らかな描画を実現。Tab文字や文字幅の厳密な処理。
  - **カーソル制御**: `DECSC` / `DECRC` シーケンス、および `DECSCUSR` (カーソル形状変更) をサポート。Block, Underline, Bar の各スタイルを動的に切り替え可能。
  - **スクロールバック**: 過去10,000行までの出力を保持し、マウスホイールやスクロールバーで閲覧可能。
- **Robust Window Management**:
  - **トランザクション的初期化**: ウィンドウ作成、ConPTY 起動、スレッド生成の各プロセスをアトミックに管理。失敗時には作成途中のリソースを確実に破棄・リセットし、ゾンビ状態を防止。
  - **二重起動とフォーカスの安全な制御**: 同一プロセス内での多重起動を論理的に防止。新規起動および再表示（二重起動試行）時に自動的にターミナルウィンドウへフォーカスを当て、既存ウィンドウの生存確認 (`IsWindow`) と組み合わせることで、クラッシュのない安定した操作感を提供。
  - **物理サイズに完全同期した起動シーケンス**: ConPTY の起動を EmEditor によるウィンドウ配置（WM_SIZE）が確定するまで遅延。フォントメトリクスの先行取得と組み合わせることで、Gemini CLI 等の起動が速い TUI アプリケーションにおいても、起動直後の表示崩れやはみ出しを完全に排除。
- **Basic Terminal I/O**: `pwsh.exe` 等のシェルとの対話が可能。
  - **入力**: キーボードフックによる直接入力。
  - **出力**: カスタムバーによる GDI 描画。
- **Config Dialog & Reflection**:
  - **GUI設定基盤**: EmEditor の「プラグインの設定」から呼び出し可能な設定ダイアログを実装。
  - **フォント構成の完全な反映**: フォント名、ポイントサイズに加え、**太字 (Weight) および 斜体 (Italic)** の設定・保存・描画反映を完全に実現。
  - **カラーテーマの外部化**: `One Half Dark/Light` テーマを選択可能にし、プラグイン設定（INI/レジストリ）への永続化を実現。
  - **システム設定への自動追従**: `System Default (Auto)` モードを搭載。Windows のダークモード設定を自動検出し、最適な配色を適用。
  - **透過的な永続化**: INIモード（ポータブル版）およびレジストリの両環境において、`Terminal` セクションへの確実な読み書きを実現。
  - **堅牢な設定ロード**: ウィンドウハンドルの適切な追跡（editor_handle）により、リサイズ中や起動時の不安定なタイミングでもユーザー設定を正確に捕捉。
  - **バリデーション・フォールバック**: 不正な設定値（0以下）が読み込まれた場合に安全なデフォルト値へ自動復帰する保護層を搭載し、描画メトリクスの破損を根絶。
- **Universal IME Positioning**:
  - **論理カーソル追跡 (Logical Cursor Tracking)**: Gemini CLI 等、描画効率のためにハードウェアカーソルを画面端へ退避（Parking）させる TUI アプリケーションにおいても、IME 候補ウィンドウが正しい入力位置に出現するインテリジェントなアンカー選択ロジックを搭載。
  - **属性ベース座標補正**: 反転属性 (SGR 7) で描画された最新位置を論理的な入力位置として動的に捕捉し、ハードウェアカーソルが非表示・退避状態にある場合でも、ユーザーの視線に一致した IME 追従を実現。
  - **UX と視認性の最適化**: `CFS_RECT` スタイルによる除外矩形設定を採用。IME 候補リストが入力中の文字を覆い隠さないよう調整し、スムーズな入力体験を提供。
  - **同期精度の向上**: 描画更新と OS 側のシステムキャレット認識を強制的に同期 (`UpdateWindow`) させることで、高速な画面更新下での「座標の置き去り」を防止。
