# Specification - Track 83: fix: edit等のアプリケーションにおける表示ずれの修正

## 1. Overview
`edit` (modern 64-bit version) 等の TUI アプリケーションにおいて、下方向へのスクロール時や特定の操作（メニュー表示等）で表示がずれる、あるいは描画が欠落・残存する不具合を修正する。
これは、ConPTY から送られてくる特定の制御シーケンス（特にスクロール領域制限 DECSTBM や、それに関連するカーソル移動）の解釈、あるいはバッファ管理の不備が原因である可能性が高い。

## 2. Functional Requirements
- **スクロール時の描画整合性**: `edit` で下方向にスクロールした際、テキスト行が正しく更新され、上部のメニューバー等が消えたり崩れたりしないこと。
- **描画の残存・欠落の解消**: 画面更新時に古い文字が残ったり、必要な罫線が消えたりする現象を修正する。
- **DECSTBM (Scrolling Region) の再検証**: 現在の実装におけるスクロール領域の扱いと、カーソル位置計算の齟齬を修正する。
- **未対応シーケンスの特定と実装**: 調査により判明した、描画ズレの原因となる未対応または誤対応の制御シーケンスを修正する。

## 3. Non-Functional Requirements
- **パフォーマンス**: 描画ロジックの修正により、スクロール速度や応答性が低下しないこと。
- **堅牢性**: 不正なシーケンスや断片化したデータを受信しても、描画が壊れないこと。

## 4. Acceptance Criteria
- `edit` (Microsoft Edit) を起動し、ファイルを下部までスクロールしても表示が崩れない。
- メニューを表示した際に、背景のテキストが正しく保持・復元される、あるいは正しく上書き描画される。
- 罫線や枠線が途切れず、正しく表示される。
- スクロール中に上部の固定領域（メニュー等）が維持される。

## 5. Out of Scope
- `edit` 以外の特定の TUI アプリにおける個別の表示崩れ（本修正の副作用で直る可能性はあるが、個別対応は本トラックの主目的ではない）。
- パフォーマンスの劇的な向上（Issue #22 ですでに対応済みのため）。
