# Specification - Track: PR #89 Review Fixes

## Overview
プルリクエスト #89 (edit 等の表示ズレ・クラッシュ修正) に対する GitHub Copilot のレビュー指摘事項を修正・改善する。

## Functional Requirements
- **安全性の向上 (Critical/Safety)**
    - `SendHFONT` 構造体における `unsafe` 実装の正当性と制約に関するドキュメントを復元する。
    - `TerminalBufferEntity` および描画ループにおける配列アクセスの境界チェックを徹底し、パニックを防止する。
    - フォント作成失敗時のフォールバックロジックを復元し、無効なハンドルによるクラッシュを防止する。
- **表示の正確性の向上 (Bug Fix)**
    - `char_display_width` において、ハングルや全角記号等の Unicode レンジを復元し、正確なカラム幅計算を維持する。
    - タブ文字処理における `cursor.x` の二重加算を修正し、カーソル移動の整合性を確保する。
    - 画面下部の残り領域および行の右側の未使用領域のクリア処理を復元し、描画の残像を防止する。
    - アクセシビリティ確保のためのコントラスト自動補正ロジックを復元・改善する。
- **品質と保守性の向上**
    - バッファ管理の変更に合わせて、削除された重要なユニットテストを復元および再実装する。

## Non-Functional Requirements
- **パフォーマンス**: 描画ループ内の境界チェック追加による性能劣化を最小限に抑える。
- **堅牢性**: 異常なエスケープシーケンスやバッファ状態でもパニックを発生させない。

## Acceptance Criteria
- 全ての GitHub Copilot レビュー指摘事項が解決、または適切な対応が行われていること。
- `edit` などの TUI アプリにおいて、表示の乱れや残像が発生しないこと。
- 復元・追加されたユニットテストが全てパスすること。
- リグレッション（既存機能の不具合）が発生していないこと。

## Out of Scope
- 今回の指摘に含まれない新しい描画機能の追加。
- `edit` 以外のアプリケーション固有の表示最適化。
