# Specification: Issue 132 - IME System Caret Sync (Clean-Slate Experimentation)

## Overview
本トラックは、GitHub Issue 132「システムキャレットの常時同期による IME 候補ウィンドウ表示位置の汎用的解決」を解決するための**実験的実装トラック**である。
対象は、通常のシェル入力ではなく、Gemini CLI 等の複雑なカーソル制御を行う TUI アプリケーション動作時の IME 候補ウィンドウの表示位置異常（0,0に固定される等）の解消である。

## Methodology (クリーン・スレート反復検証)
過去の試行錯誤による「コードの汚染（誤った修正の上にさらに修正を重ねることによる迷走）」を防ぐため、以下の厳格なワークフローを適用する。
1. **基盤準備**: 検証を容易にするための最小限のデバッグ機構（ログ出力や特定の検証用コマンド等）のみを実装し、その状態を「クリーンな起点」とする。
2. **仮説立案**: `evidence_report.md` に仮説（解決策A）を記述する。
3. **実験的実装**: 仮説に基づくコードを実装する。
4. **検証**: Gemini CLI を起動して IME の挙動を確認する。
5. **リセットの義務**: 検証が失敗した場合、**直ちに `src/` 配下の変更を破棄（`git reset --hard` 等でメインブランチの状態、または基盤準備の状態へ戻す）**し、ステップ2（仮説B）へ戻る。

## Scope
1. TUI アプリケーション動作時の仮想カーソル移動に伴う、OS のシステムキャレット（`SetCaretPos`）の同期ロジックの確立。
2. 同期されたシステムキャレット位置に基づく、正確な IME 候補ウィンドウ位置（`ImmSetCompositionWindow`, `CFS_RECT` / `CFS_POINT`）の計算と設定。

## Functional Requirements
1. **仮説と結果の記録**: 各実験サイクルの仮説と結果を `evidence_report.md` に追記していくこと。
2. **最終実装**: 全ての検証に合格した「唯一の正しい解決策」のみを、クリーンな状態のコードベースに適用し、完成させること。

## Acceptance Criteria
1. Gemini CLI 等の TUI アプリケーション上で日本語入力を行う際、IME 候補ウィンドウが現在の仮想カーソル位置に正確に追従して表示されること。
2. 通常のシェル入力時の IME 挙動が破壊されていないこと。
3. 最終的なコードが、過去の失敗した実験コードを含まない、クリーンな差分となっていること。

## Out of Scope
1. IME の表示位置以外の、ターミナルエミュレータの既存機能の大規模なリファクタリング。
