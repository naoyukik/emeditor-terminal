# Specification: Issue #22 描画の最適化とちらつき防止 (GDI ダブルバッファリング)

## Overview
現在、Gemini CLIなどのTUIアプリケーション実行時に描画の乱れやちらつき（Flickering）が発生している。
この問題を解決するため、GDI描画においてダブルバッファリング（Memory DC / Compatible Bitmap）を導入し、描画パイプラインを最適化する。

## Functional Requirements
1.  **描画のダブルバッファリング化**:
    -   `TerminalGuiDriver::render` 内部でメモリDCおよび互換ビットマップを作成し、すべての描画処理（テキスト、背景、カーソル、IME）をオフスクリーンで行う。
    -   描画完了後、`BitBlt` を使用してウィンドウDCへ一括転送する。
2.  **背景消去の抑制**:
    -   `WM_ERASEBKGND` メッセージをハンドルし、`1` (TRUE) を返すことで、OSによるデフォルトの背景消去（白塗り等）を防止する。
3.  **リソース管理**:
    -   作成したメモリDCおよびビットマップを適切に解放し、GDIオブジェクトのリソースリークを防止する。

## Non-Functional Requirements
-   **パフォーマンス**: 実装の単純なオンデマンド方式から開始し、必要に応じてリサイズ時のキャッシュを検討する。
-   **安定性**: ウィンドウリサイズ時や、描画バッファが一時的に空の場合でも、クラッシュや描画の乱れが発生しないこと。

## Acceptance Criteria
-   [ ] 高速なターミナル更新時に、画面のちらつきが視認できないこと。
-   [ ] ウィンドウのリサイズ中に、描画内容が崩れたり、残像が残ったりしないこと。
-   [ ] メモリ消費量やCPU負荷に顕著な増加が見られないこと。
