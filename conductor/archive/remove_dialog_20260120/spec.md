# 仕様書: 起動時の入力ダイアログを廃止し直接ターミナルを起動する

## 1. 概要
現在、EmEditorプラグインのボタンを押下した際、コマンド入力を求めるダイアログが表示される。本トラックではこのダイアログを完全に廃止し、即座にデフォルトシェルでターミナルを起動するように変更する。

## 2. 機能要件
- **ダイアログの廃止**: `src/dialog.rs` による入力ダイアログ表示を削除する。
- **デフォルトシェルの起動**: ダイアログからの入力に代わり、常に `pwsh.exe` (PowerShell 7) を起動する。
- **起動フローの変更**: `OnCommand` が呼ばれた際、`custom_bar::open_custom_bar` を呼び出した後、ユーザー入力を待たずにターミナルを初期化/表示する。
- **既存コードへの影響**: `src/dialog.rs` ファイルおよびそれに関連するモジュール定義をプロジェクトから削除する。

## 3. 非機能要件
- **レスポンス性**: ボタン押下からターミナルが利用可能になるまでの待機時間を最小限にする。

## 4. 受け入れ基準 (Acceptance Criteria)
- [ ] EmEditorでプラグインボタンを押した際、ダイアログが表示されないこと。
- [ ] `pwsh.exe` がターミナル内で正しく起動すること。
- [ ] プロジェクトから `src/dialog.rs` が削除されており、コンパイルエラーが発生しないこと。

## 5. 範囲外 (Out of Scope)
- シェルパスをユーザーが設定可能にする機能（本トラックでは `pwsh.exe` 固定とする）。
- 複数のカスタムバー管理（タブ化）の内部ロジック詳細変更（EmEditor側の挙動に依存）。
